# アプリ名
ひみつQR

# コンセプト
「ひみつQR」は、あいことばを知っている人だけが読める秘密のQRメッセージを作成できるエンタメ向けWebアプリです。

ユーザーは秘密のメッセージ、あいことば、あいことばのヒント、入力形式を設定します。
アプリはメッセージをあいことばで暗号化し、QRコードを生成します。

受け取った人はQRコードを読み取り、ヒントと入力形式を見て、あいことばを入力します。
正しいあいことばなら、秘密のメッセージが表示されます。

これは医療情報、金融情報、業務機密などを扱う本格セキュリティツールではなく、プレゼント、謎解き、推し活、SNS遊び、イベント演出向けのプロトタイプです。

# 技術スタック
- Vite

- CSS Modules または通常のCSS

# 重要な方針
- 独自暗号は作らない
- Base64変換だけを暗号化と呼ばない
- Web Crypto APIのAES-GCMを使う
- あいことばからPBKDF2-SHA256でAES鍵を導出する
- saltとivは暗号化ごとにcrypto.getRandomValuesでランダム生成する
- 本文とあいことばは保存しない
- localStorageにあいことばや本文を保存しない
- ヒント、入力形式、文字数ヒントはQRを読み取った人なら見える公開情報として扱う
- エンタメ用途であることを画面上にも明記する

# 暗号仕様
暗号化方式：
- AES-GCM
- AES鍵長：256bit
- KDF：PBKDF2-SHA256
- PBKDF2 iterations：300000
- salt：16 bytes
- iv：12 bytes

暗号化の流れ：
1. ユーザーが秘密メッセージ、あいことば、ヒント、入力形式を入力
2. saltをランダム生成
3. ivをランダム生成
4. あいことば + salt からPBKDF2-SHA256でAES-GCM用の鍵を生成
5. 秘密メッセージをAES-GCMで暗号化
6. 以下のJSONを作成
   {
     "v": 1,
     "app": "himitsu-qr",
     "alg": "AES-GCM",
     "kdf": "PBKDF2-SHA256",
     "iter": 300000,
     "salt": "...base64...",
     "iv": "...base64...",
     "hint": "...",
     "format": "...",
     "showLength": true,
     "length": 5,
     "ciphertext": "...base64..."
   }
7. JSONをBase64URL化する
8. QRコードには以下のようなURL形式で格納する
   現在のURL + "#open=" + encodedPayload

例：
http://localhost:5173/#open=xxxxx

復号の流れ：
1. URLのhashからopenパラメータを読み取る
2. payloadをBase64URLデコードしてJSONを復元
3. ヒント、入力形式、文字数ヒントを表示
4. ユーザーがあいことばを入力
5. QR内のsaltと入力されたあいことばから同じ鍵を再生成
6. ivとciphertextでAES-GCM復号
7. 成功したら本文表示
8. 失敗したら「開けませんでした」と表示

# 画面構成

## 1. ホーム兼作成画面
スマホ縦画面で見やすいUIにしてください。

表示するコピー：
- あいことばで開く、秘密のQRメッセージ。
- プレゼント、謎解き、推し活、イベントに。
- QRを読み取っても、あいことばを知らないと中身は読めません。

入力欄：
1. 秘密のメッセージ
   - textarea
   - placeholder: 例）誕生日おめでとう。いつもありがとう。

2. あいことば
   - passwordまたはtext切り替え可能
   - placeholder: 例）カフェルナ

3. あいことば確認
   - passwordまたはtext切り替え可能
   - あいことばと一致しない場合は警告

4. あいことばのヒント
   - input
   - placeholder: 例）初めて一緒に行ったカフェの名前
   - 補足文：ヒントはQRを読み取った人なら誰でも見られます。

5. 入力形式
   - select
   - 選択肢：
     - 指定なし
     - ひらがなのみ
     - カタカナのみ
     - 漢字あり
     - 数字のみ
     - 英字のみ
     - 英数字
     - 記号あり

6. 文字数ヒントを表示する
   - checkbox
   - ONの場合、あいことばの文字数を復号画面に表示する
   - 補足文：便利ですが、推測されやすくなる場合があります。

ボタン：
- 秘密QRを作る
- 入力をクリア

バリデーション：
- 秘密メッセージが空なら作成不可
- あいことばが空なら作成不可
- あいことば確認が一致しないなら作成不可
- あいことばが4文字未満なら強い警告
- あいことばが8文字未満なら注意
- 数字のみで短い場合は注意
- ヒントにあいことばそのものが含まれている場合は警告

安全度表示：
あいことばと設定内容から、ざっくり以下を表示してください。

- 安全度：低い
  条件例：4文字未満、数字のみ、ヒントに答えが含まれている
- 安全度：ふつう
  条件例：8文字以上
- 安全度：高め
  条件例：16文字以上、数字のみではない、ヒントに答えが含まれていない

ただし、文言はエンタメ向けにしてください。
例：
「サプライズ用途ならOK」
「誰でも推測できそうです」
「かなり開けにくそうです」

## 2. QR生成結果画面
QR生成後に表示する。

表示内容：
- 秘密QRができました
- QRコード画像
- このQRには暗号化されたメッセージが入っています
- あいことばを知っている人だけが開けます

ボタン：
- QR画像を保存
- QRのURLをコピー
- 暗号データをコピー
- もう一度作る

実装：
- QR画像保存は canvas または dataURL からPNG保存できるようにする
- URLコピーは navigator.clipboard.writeText を使う
- 失敗時は手動コピー用にURLを表示する

注意：
QRコードが長くなりすぎる場合があります。
秘密メッセージが長すぎる場合は、以下のような注意を出してください。
「文章が長いとQRコードが読み取りにくくなる場合があります。短めのメッセージがおすすめです。」

## 3. 開封画面
URLのhashに #open= がある場合、自動的に開封画面を表示する。

表示内容：
- 秘密のメッセージが届いています
- ヒント
- 入力形式
- 文字数ヒントがONの場合だけ文字数
- あいことば入力欄
- 開くボタン

画面例：
秘密のメッセージが届いています

ヒント：
初めて一緒に行ったカフェの名前

入力形式：
カタカナのみ

文字数：
5文字

あいことばを入力してください

[　　　　　]

[開く]

復号成功時：
- メッセージが開きました
- 本文を表示
- コピーするボタン
- 最初に戻るボタン

復号失敗時：
- 開けませんでした
- あいことばが違うか、QRコードの内容が壊れている可能性があります。
- ひらがな、カタカナ、漢字、数字、スペース、記号が同じか確認してください。

# 各ファイルの役割

## lib/crypto.ts
- deriveKey(passphrase, salt, iterations)
- encryptMessage(message, passphrase)
- decryptMessage(payload, passphrase)
- AES-GCMとPBKDF2の処理をここに集約
- TextEncoder / TextDecoder を使う
- Uint8ArrayとBase64URLの変換はencoding.tsに任せる

## lib/encoding.ts
- bytesToBase64
- base64ToBytes
- base64UrlEncodeString
- base64UrlDecodeString
- payloadToEncoded
- encodedToPayload

## lib/qr.ts
- QRコード生成関連
- qrcodeライブラリを使う
- generateQRCodeDataUrl(text)

## lib/validation.ts
- あいことば強度チェック
- ヒントに答えが含まれるかのチェック
- 入力チェック

## types/secretPayload.ts
SecretPayload型を定義する

例：
export type SecretPayload = {
  v: number;
  app: "himitsu-qr";
  alg: "AES-GCM";
  kdf: "PBKDF2-SHA256";
  iter: number;
  salt: string;
  iv: string;
  hint: string;
  format: string;
  showLength: boolean;
  length?: number;
  ciphertext: string;
};
